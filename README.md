## Abstract
```
With the rapid rise of food delivery platforms and dining culture, making healthy food choices has become increasingly difficult. According to the World Health Organization (WHO), 43% of adults worldwide are overweight, and obesity rates have doubled since 1990. A major contributing factor is the lack of actionable nutritional information in restaurant menus.

This project proposes a Smart Nutrition-Based Restaurant Filtering System that helps users make informed food choices based on personalized health goals such as weight loss, muscle gain, or disease-specific diets. Unlike existing platforms that provide static calorie information, this system introduces dynamic filtering and recommendation features using nutritional data, user profiles, and decision-based logic.

The system allows users to filter meals based on calories, macronutrients, and dietary preferences while also providing intelligent suggestions such as “best meals under 500 calories” or “high-protein options.” The expected outcome is improved decision-making, reduced unhealthy food consumption, and enhanced user awareness. This project bridges the gap between nutrition awareness and real-world food ordering behavior.
```

## introduction
```
The increasing dependence on restaurant and online food delivery services has significantly impacted dietary habits. Studies show that individuals consume approximately 200 extra calories per day when eating outside, contributing to long-term health risks. Furthermore, over 60% of restaurant meals exceed recommended calorie limits, making it difficult for users to maintain a healthy diet.

Despite growing awareness about fitness and nutrition, users face challenges in interpreting complex nutritional information. Existing food platforms focus primarily on convenience and taste rather than health, offering minimal filtering options such as vegetarian or non-vegetarian categories.

This project aims to address this gap by developing a smart filtering and recommendation system that simplifies decision-making. The objective is to transform static menu data into actionable insights, enabling users to choose meals aligned with their health goals.

“People don’t choose unhealthy food intentionally—they choose it because healthier options are harder to identify.”
```
## Literature review
```
Several studies and applications have attempted to improve dietary choices through nutritional awareness. Applications like MyFitnessPal focus on calorie tracking but require manual input and do not integrate with restaurant menus. Food delivery platforms like Zomato and Swiggy provide limited filtering options and lack deep nutritional insights.

Research indicates that calorie labeling alone has minimal impact, reducing intake by only around 11 calories per meal. Additionally, only 31% of users actively use nutrition labels, highlighting the need for more intuitive systems.

Studies also show that the local food environment influences 10–15% of obesity variation, emphasizing the importance of accessible healthy choices. Existing solutions fail to provide personalized, real-time recommendations, which is a key gap addressed by this project.

This system builds upon existing research by combining nutrition data, personalization, and decision-support mechanisms into a unified platform.
```

## Methodology
```
The system collects restaurant menu data and maps it to nutritional datasets such as calories, protein, fats, and carbohydrates. Users create profiles by entering their health goals and preferences. A filtering engine processes this data to display suitable food options. Additionally, a recommendation module suggests optimal meals based on user requirements. The system prioritizes simplicity and usability by presenting only the most relevant choices instead of overwhelming users with raw data.
```
